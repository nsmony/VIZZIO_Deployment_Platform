Project Overview
Build a
Steam-style deployment system that lets authorized users download and install digital twin
deployments on their machines Each deployment is a self-contained package (an Unreal
executable folder; two web folders; and a batch script that we provide): Multiple versions of each
deployment must be able to coexist on the same machine so users can keep both stable and beta
releases:
The system has two products: a web admin panel for managing users and deployments, and
a
lightweight Windows launcher app that handles downloads and lets users open the install folder:
The brief below describes what the system should do. The team is expected to propose the technology choices and
architecture in Week 1 and present them for sign-off
What the System Does
Admin Web Panel
Admin logs in with a username and password.
Create; edit; disable; and reset passwords for users:
Create deployments and add new versions to each deployment;
Mark each version as either stable or beta:
Mark each version as released (visible to users) or archived (hidden):
Control which users can see which deployments
Adding
a new version: admin places the deployment folder on the server; then triggers an
Add version" action in the panel:
Launcher Client
Simple; clean design: White background. Logo is swappable so we can rebrand it per client:
User logs in with the same username and password.
Sees only the deployments they have been
access to.
Sees every released version of each deployment; grouped by stable and beta.
Downloads run fast
files come in via multiple parallel streams
